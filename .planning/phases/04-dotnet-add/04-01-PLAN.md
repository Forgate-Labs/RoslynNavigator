  <behavior>
    Method signature:
      `AddMemberResult AddMember(string sourceText, string typeName, string memberKind, string content)`
      `AddUsingResult AddUsing(string sourceText, string namespaceName)`

    AddMemberResult record:
      `bool Success`
      `string ModifiedSource`   // full modified file content
      `string? Error`           // null on success

    AddUsingResult record:
      `bool Success`
      `string ModifiedSource`
      `bool AlreadyPresent`     // true when using was already there (no-op success)
      `string? Error`

    member kind values: "field" | "property" | "constructor" | "method"
    target types: ClassDeclarationSyntax, RecordDeclarationSyntax, StructDeclarationSyntax

    Insertion position rules (DADD-06):
      - field    → after last existing field; if no fields exist, first inside type body
      - property → after last existing property; if no properties, after last field; if no fields, first
      - constructor → after last property; if no properties, after last field; if no fields, first
      - method  → before closing brace of type body (last position)
      - using   → inserted in sorted order among existing using directives; no-op if already present

    Indentation detection:
      - Scan existing members of the target type to find leading whitespace of the first member
      - If no existing members, use 4 spaces (default)
      - Apply detected indentation to each line of the inserted content

    Syntax validation (DADD-05):
      - For field/property/constructor/method: wrap content in a temp class, parse with CSharpSyntaxTree.ParseText,
        check tree.GetDiagnostics() for errors — if any error exists, return AddMemberResult { Success=false, Error="..." }
      - For using: no syntax validation needed (namespace name is a simple string)

    Test cases:
      - AddMember(classSource, "MyClass", "field", "private int _count;") → field appears after last field
      - AddMember(classSource_noFields, "MyClass", "field", "private int _x;") → field appears first in body
      - AddMember(classSource_withFields, "MyClass", "property", "public int Count { get; set; }") → after fields
      - AddMember(classSource_withAll, "MyClass", "constructor", "public MyClass() { }") → after properties
      - AddMember(classSource_withAll, "MyClass", "method", "public void Do() { }") → before closing brace
      - AddMember(structSource, "MyStruct", "field", "private int _x;") → works for struct
      - AddMember(recordSource, "MyRecord", "property", "public int Id { get; init; }") → works for record
      - AddMember(classSource, "NonExistent", "method", "public void X() { }") → Error = "Type 'NonExistent' not found"
      - AddMember(classSource, "MyClass", "method", "NOT VALID C# {{{{") → Error contains "syntax error"
      - AddUsing(source_noUsing, "System.Collections.Generic") → using added at top
      - AddUsing(source_withUsing, "System") → AlreadyPresent = true, source unchanged
  </behavior>

  <implementation>
    Create `DotnetAddMemberService` as a static class in `src/RoslynNavigator/Services/DotnetAddMemberService.cs`.

    Use `Microsoft.CodeAnalysis.CSharp` (already a project dependency) — do NOT add new NuGet packages.

    Key Roslyn APIs:
      - `CSharpSyntaxTree.ParseText(sourceText)` — parse the file source
      - `root.DescendantNodes().OfType<BaseTypeDeclarationSyntax>()` — find all class/record/struct nodes
        (ClassDeclarationSyntax, RecordDeclarationSyntax, StructDeclarationSyntax all extend BaseTypeDeclarationSyntax)
      - Filter by `typeDecl.Identifier.Text == typeName`
      - Member ordering: iterate `typeDecl.Members` to categorize as FieldDeclarationSyntax,
        PropertyDeclarationSyntax, ConstructorDeclarationSyntax, MethodDeclarationSyntax
      - Insertion via `typeDecl.WithMembers(typeDecl.Members.Insert(index, parsedMember))`
        where parsedMember = `(MemberDeclarationSyntax)CSharpSyntaxTree.ParseText("class T { " + content + " }").GetRoot().DescendantNodes().First(n => n is MemberDeclarationSyntax)`
      - Indentation: use `typeDecl.Members.FirstOrDefault()?.GetLeadingTrivia().ToString()` or detect
        from first non-whitespace member; fall back to "    " (4 spaces)
      - Apply indentation to each line of content before parsing
      - For using: `root.Usings` contains `UsingDirectiveSyntax` nodes; check if any `.Name.ToString() == namespaceName`;
        insert sorted if not found using `root.WithUsings(root.Usings.Add(SyntaxFactory.UsingDirective(...)))`
        (sort alphabetically, or append at end — either is acceptable)
      - Return `root.ToFullString()` as ModifiedSource

    Syntax validation approach:
      Wrap content: `var testSrc = $"class __Test__ {{\n{content}\n}}"`;
      `var testTree = CSharpSyntaxTree.ParseText(testSrc)`;
      `var diags = testTree.GetDiagnostics().Where(d => d.Severity == DiagnosticSeverity.Error)`;
      If any error → return failure result.
  </implementation>
