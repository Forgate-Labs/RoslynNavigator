**1. Create `src/RoslynNavigator/Models/DotnetScaffoldResults.cs`**

File-scoped namespace `RoslynNavigator.Models`. Single record:

```csharp
namespace RoslynNavigator.Models;

public record DotnetScaffoldResult
{
    public required string Operation { get; init; }   // "scaffold-class" | "scaffold-interface" | "scaffold-record" | "scaffold-enum"
    public required string FilePath { get; init; }
    public required string TypeName { get; init; }
    public required string Namespace { get; init; }
    public int TotalStagedOps { get; init; }
}
```

**2. Create `src/RoslynNavigator/Commands/DotnetScaffoldCommand.cs`**

File-scoped namespace `RoslynNavigator.Commands`. Static class with a single `ExecuteAsync` method. It:
- Accepts `(string path, string ns, string typeName, string scaffoldType)` where `scaffoldType` is one of `"class"`, `"interface"`, `"record"`, `"enum"`
- Generates the C# file content as a string using the file-scoped namespace pattern. Templates:
  - class: `"namespace {ns};\n\npublic class {typeName}\n{\n}\n"`
  - interface: `"namespace {ns};\n\npublic interface {typeName}\n{\n}\n"`
  - record: `"namespace {ns};\n\npublic record {typeName}\n{\n}\n"`
  - enum: `"namespace {ns};\n\npublic enum {typeName}\n{\n}\n"`
- Creates a `PlanOperation` with `Type = OperationType.ScaffoldFile`, `FilePath = path`, `NewContent = <generated content>`
- Loads plan state via `FilePlanStore.CreateDefault()`, adds the op, saves, returns `DotnetScaffoldResult`

The `Operation` string in the result is `"scaffold-{scaffoldType}"` (e.g. `"scaffold-class"`).

**3. Extend `src/RoslynNavigator/Services/FilePlanEngine.cs` — `ApplyOpsInMemory` method**

In the `switch (op.Type)` block inside `ApplyOpsInMemory`, add a case for `OperationType.ScaffoldFile` that behaves identically to `OperationType.Write` — clears the file lines and splits `NewContent` on `'\n'`, trimming `'\r'` from each line. This ensures scaffold ops appear in `file status` diffs and are applied by `file commit`.

Also add `OperationType.ScaffoldFile` to the `ValidateAsync` skip list (alongside Write and Append) — scaffold ops require no pre-validation since they always create/overwrite.

    record (Command) — args: path, namespace, recordName
    enum (Command) — args: path, namespace, enumName

**For each of the four scaffold subcommands:**
- Three positional `Argument<string>`: `path`, `namespace`, `typeName` (use descriptive arg names like `classNameArg`, `interfacePathArg` etc. to avoid conflicts)
