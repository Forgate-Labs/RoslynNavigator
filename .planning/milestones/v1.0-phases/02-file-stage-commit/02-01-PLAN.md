  <behavior>
    FilePlanEngine is a stateless service (or near-stateless — it receives deps via constructor).
    It exposes these public methods:

    1. `ValidateAsync(IReadOnlyList&lt;PlanOperation&gt; ops, string workingDir)`
       Returns `IReadOnlyList&lt;string&gt; errors` (empty = valid).
       For Edit and Delete ops: reads file at op.FilePath (absolute or relative to workingDir),
       checks that line op.Line contains op.OldContent. If not → error message:
         "Edit op on '{path}' line {n}: expected '{old}' but found '{actual}'"
         "Delete op on '{path}' line {n}: expected '{old}' but found '{actual}'"
       File not found → "File not found: {path}"
       Line out of range → "Line {n} is out of range in '{path}' (file has {total} lines)"

    2. `ComputeDiff(IReadOnlyList&lt;PlanOperation&gt; ops, string workingDir)`
       Returns `string unifiedDiff`.
       Applies ops in order to in-memory copies of the files, produces unified diff format:
         --- a/path/to/file
         +++ b/path/to/file
         @@ -start,len +start,len @@
         context/changed lines prefixed with space/ -/ +
       Uses 3 lines of context. Groups hunks by file (multiple ops on same file merge into one diff block).
       Write/Append ops: treat original as empty if file doesn't exist.
       Delete op: remove the line (shift subsequent line numbers).

    3. `CommitAsync(IReadOnlyList&lt;PlanOperation&gt; ops, string workingDir, BackupService backupService)`
       Returns `ApplyResult { string UnifiedDiff; string BackupPath }`.
       Steps:
         a. Validate all ops (call ValidateAsync). If any errors → throw InvalidOperationException listing all errors.
         b. Create backup of all affected files via backupService.CreateBackupAsync.
         c. Apply all ops to disk atomically (compute final file states in memory first, then write all files).
         d. Compute unified diff (between backup state and new state).
         e. Return ApplyResult with the diff and backup path.

    4. `RollbackAsync(string backupPath, string workingDir)`
       Reads all files in backupPath directory tree, copies them back to workingDir preserving relative paths.
       If backupPath doesn't exist → throw InvalidOperationException: "Backup not found: {backupPath}"

    Cases for tests (RED before GREEN):
      - Edit op where line matches → ValidateAsync returns empty errors
      - Edit op where line does NOT match → ValidateAsync returns error with exact message
      - Edit op where file does not exist → "File not found" error
      - Edit op where line is out of range → "Line N is out of range" error
      - Delete op where line matches → ValidateAsync returns empty errors
      - Delete op where line does NOT match → error
      - Write op (no validation needed) → ValidateAsync returns empty errors even if file missing
      - Append op (no validation needed) → ValidateAsync returns empty errors even if file missing
      - CommitAsync with all-valid ops → writes files, returns non-empty diff and backup path
      - CommitAsync with invalid op → throws, no files written
      - RollbackAsync with valid backupPath → files restored to original content
      - RollbackAsync with missing backupPath → throws InvalidOperationException
  </behavior>
  <implementation>
    File-scoped namespace: `namespace RoslynNavigator.Services;`
    Use `public class FilePlanEngine` (not static — needs BackupService dep).
    `ApplyResult` is a record: `public record ApplyResult(string UnifiedDiff, string BackupPath);`
    Place ApplyResult in `src/RoslynNavigator/Services/FilePlanEngine.cs` (same file, before the class).

    For unified diff computation: implement a minimal Myers diff or line-by-line diff yourself (no external library).
    The format must be parseable by standard tools but does not need to be git-identical.
    Minimum: each changed hunk prefixed with @@ header, removed lines with `-`, added lines with `+`, 3-line context.

    Apply operations in-memory before writing:
      - Build a Dictionary&lt;string, List&lt;string&gt;&gt; of file → lines (read from disk or empty for Write/new Append).
      - Apply each op in order: Edit replaces line[op.Line-1]; Write replaces entire list; Append adds at end; Delete removes line[op.Line-1].
      - After all ops applied in memory, write each modified file to disk.
      - This guarantees atomicity-of-consistency: all reads happen before any write.

    Test project: check if `tests/RoslynNavigator.Tests/` exists. If not, create it as xUnit project and add to solution.
    Use `System.IO.Abstractions` if already in project; otherwise use real filesystem with temp directories in tests.
    Tests should use `Path.GetTempPath()` + `Guid.NewGuid()` for isolated temp directories, cleaning up in Dispose.
  </implementation>
