    Static class following existing pattern (see FileGrepCommand.cs for reference).

    ```csharp
    public static class FilePlanEditCommand
    {
        public static async Task<FilePlanStagedResult> ExecuteAsync(
            string path, int line, string oldContent, string newContent)
    ```

    Steps inside ExecuteAsync:
    1. Create a temporary single-op list: `new PlanOperation { Type = OperationType.Edit, FilePath = path, Line = line, OldContent = oldContent, NewContent = newContent }`.
    2. Call `new FilePlanEngine().ValidateAsync([op], Directory.GetCurrentDirectory())`.
    3. If errors is non-empty → throw `InvalidOperationException(errors[0])` (first error wins; CROSS-02 specific messages come from engine).
    4. Load plan state via `FilePlanStore.CreateDefault().LoadAsync()`.
    5. Add the op to state.Operations.
    6. Save via `FilePlanStore.CreateDefault().SaveAsync(state)`.
    7. Return `new FilePlanStagedResult { Operation = "edit", FilePath = path, Line = line, TotalStagedOps = state.Operations.Count }`.

    Create `src/RoslynNavigator/Commands/FilePlanDeleteCommand.cs` with same pattern:
    ```csharp
    public static class FilePlanDeleteCommand
    {
        public static async Task<FilePlanStagedResult> ExecuteAsync(
            string path, int line, string oldContent)
    ```
    Same steps but OperationType.Delete, no NewContent, Operation = "delete".
