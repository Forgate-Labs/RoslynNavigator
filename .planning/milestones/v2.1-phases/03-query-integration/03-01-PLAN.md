  <action>
Create `SqlReadOnlyGuard` as a dedicated service that validates whether SQL is read-only and safe to run against snapshot databases.

Guard rules must reject: empty SQL, SQL containing statement separators (`;` with additional statements), and any mutating/DDL keywords (e.g., INSERT, UPDATE, DELETE, CREATE, DROP, ALTER, REPLACE, PRAGMA, ATTACH, DETACH, VACUUM). Allow only queries that start with `SELECT` or `WITH` after trimming whitespace/comments.

Return structured validation output (`isValid`, `reason`) so command/services can produce actionable errors.
  </action>
  <verify>
